Treetables are for organizing hierarchical data with multiple columns in a datagrid.

Use When

Use this component when you need to display a hierarchichal data structure with multiple columns per entry. Make sure all entries use the same columns. E.g. the parent has a name and a description column, the child cannot have a name and a date column.

Do not use

Avoid using this component in any of the following cases:

  • You require using a pagination. The treetable does not support paging.
  • The entries in your datastructure do not have multiple values (columns). Consider using a TreeView.
  • Your elements need to be sorted. Since the data-rows are indented, a change of the order would cause confusion.

Behavior

Each entry can have none, one or multiple descendants. The first column of a descendant-row is indentend to visualize the hierarchy. Parent rows are collapsed, i.e. all children rows are hidden initially. To display the children the user can either click on the leading caret-icon or the whole parent row, depending on the row-selection mode. See the official clarity documentation of the datagrid for details of the selection mode. Avoid the rowselection-mode when the row contains individually clickable items.

Sorting

The treetable cannot be sorted by the user. Display the entries in alphabetical order. Consider collapsing either all or none of the parent nodes.

Optional: column headers

A treetable can display column headers, but does not require them.

Technical limitations

The Treetable is technically related to the datagrid, but omits lazy-loading, sorting and filtering. Note that up to 6 levels of nesting are supported.

Optional: disable horizontal scrolling

For tables with a small number of columns but with a lot of text, you might want to prevent horizontal scrolling at all. Tables usually scroll horizontally on small screens. Use the clr-treetable-noscroll class to deactivate this behavior. A table with this class always displays all columns and cuts off overflowing text with (...) aka. ellipsis. Beware that enabling this option might lead to styling issues on tables with a large number of columns and/or deeper nesting.
