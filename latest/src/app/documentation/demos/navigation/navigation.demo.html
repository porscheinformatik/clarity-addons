Content Panel

The Content Panel provides additional, in most cases contextual information to the content-area.

Responsive Bahavior

The sidebar uses on mobile devices the full page width. It is displayed as modal layer above the main content on larger screens. On medium sized desktops and above, the sidebar is a column next to the content. Note that it is always possible to hide or display the panel.

Usage

  • Use to display supplemental content and features

Behavior

  • The content panel is not always visible, it appears on the right side next to the content-area and can be shown or hidden according to users needs.

Responsive Behavior

≥1200 px (Desktop, fullscreen)

  • Content panel is displayed on the right-side next to the content-area
  • The content panel remains visible while the user interacts with the primary content.
  • According to business needs the content panel can be shown automatically on resize or on user interaction, depending whether the content area or the content panel poses more important information to the user.

≥768 px (Tablet, landscape)

  • The content panel is displayed on the right-side overlapping the content-area and invoking a backdrop over the content-area.
  • Invoking and

default: ≤ 768px (Tablet, portrait)

  • The content panel is shown maximized, consuming the content-area's space.

When implementing a content panel on your page, you will want to replace your default container div with a clr-content-panel-container. Inside of it, you can place any content you want to - including one or multiple clr-content-panel.

You can use the @ViewChild() annotation to get a reference of your ClrContentPanel, which will allow you to call its methods:

  • open() - opens the content panel.
  • close() - closes the content panel.
  • toggle() - convenience method, either opens or closes the content panel depending on the current state of it.
  • isOpen() - returns true if the content panel is open.

Following inputs & outputs are available:

  • [clrDirection] - determines if the content panel is shown on the left or right side. Default is right.
  • (clrClosed) is fired every time the content panel is closed.
  • (clrOpened) is fired every time the content panel is opened.
  • Adding any HTML with the attribute clr-content-panel-title will set the title of the content panel.
  • Adding any HTML with the attribute clr-content-panel-content will set the content of the content panel.

Main Navigation

The Main Navigation consists of a classic mainmenu bar with dropdown submenus.

Design Guidelines

The Main Navigation is an additional Navigation pattern, it enhances Clarity's Header Nav with dropdown submenus.

Behavior

  • A Header Navigation item can either lead directly to a page, or contain a submenu dropdown that leads to a list of navigation links.
  • First and second navigation level are covered in the Main Navigation, responsive accessible in the hamburger-menu on the left side.
  • Clarity supports responsive navigation for two levels of navigation.
  • Main Navigation adds a third responsive navigation level (see Sidebar Pagelayout)

Use when

  • Your application is commonly used on desktop devices
  • You have a dense information hierarchy, and need more than two levels of navigation responsive accessible

Don't use when

  • You are building a MobileFirst application with a fairly flat information hierarchy
    • Consider using Header Nav
    • Consider using Material Design for a MobileFirst approach: Bottom-navigation or navigation-drawer should be preferred.
  • Don't combine Main Navigation with Subnav Pattern.
