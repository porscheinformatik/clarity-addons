The use of breadcrumbs have been widely discussed in several ui/ux communities. Some users stick to them, others don't even recognize them.
For complex business applications, providing a clear hierarchical structure (with parent-child relation between pages, or probably deeply nested) the use can be justified.
Avoided should be solutions with historical breadcrumbs, as they are very seldom - may lead to cycles and basically duplicate the browser's back functionality.
  1. Page before last
  2. Last page
  3. Actual Page
Don't

Don't use it as browsing history.

Do

Use it as hierarchical path to the actually viewed page.

Code & Examples

To set the breadcrumbs, the service ClrBreadcrumbService in conjunction with ClrBreadcrumbModel has to be used.
