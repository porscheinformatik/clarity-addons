Design Guidelines

The use of breadcrumbs have been widely discussed in several ui/ux communities. Some users stick to them, others don't even recognize them.

For complex business applications, that provide a clear hierarchical structure (with parent-child relation between pages, which are probably deeply nested) the use can be justified.

Avoided should be solutions with historical breadcrumbs, since they are very seldom used, may lead to cycles inside the breadcrumb-trail and basically duplicate the browser's back functionality.

Use when

Your application provides a clear hierarchical structure.
The site structure is deeply nested and the primary navigation scheme is not sufficient.

Don't use when

Your information architecture represent a network instead of a hierarchical tree of pages.
In that case you can not clearly name the current page's parent, and the breadcrumb trail can not represent the path the user has been taken through the application.

Do

Use breadcrumbs as hierarchical path to the current displayed page.

  1. Page before last
  2. Last page
  3. Current Page
Don't

Don't use historical breadcrumbs as browsing history.
