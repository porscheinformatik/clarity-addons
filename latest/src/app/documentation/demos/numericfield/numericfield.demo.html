The Numeric Field is used to enter numeric values of a certain type.

Usage

  • Invalid characters are ignored during input.
  • Amount of decimal places should be defined according to the input type and usecase.
  • Decimal separator and grouping separator should be set according to locale.

Use when

  • One of the following inputs is required from the user:

Code & Examples

Because the numeric field is a directive, it can be applied on any input element. This makes the numeric field compatible with the existing ClrInputContainer and forms. See code examples below which include clr-control-error and clr-control-helper elements.

The numeric field supports template driven as well as reactive forms, see the form demos below.

The numeric field requires you to use custom validators to validate min and max values:

  • Reactive: ClrNumericFieldValidators
  • Template driven: clrMaxNumeric and clrMinNumeric

Angular Directive

Summary of Options

Input Values Default Effect
[clrTextAlign]
Type: String
Default: "right"
 left, right "right" Defines the alignment of the text inside input field.
[clrDecimalSep]
Type: String
Default: ","
 Javascript String "," Defines the decimal separator.
[clrGroupingSep]
Type: String
Default: "."
 Javascript String "." Defines the grouping separator.
[clrDecimalPlaces]
Type: Number
Default: 2
 Javascript Number 2 Defines the number of allowed decimal places.
[clrAutofillDecimals]
Type: Boolean
Default: false
 true, false false Defines whether decimal places should automatically be filled with zeroes until clrDecimalPlaces is reached. Useful for money fields.
[clrUnit]
Type: string
Default: ""
 Javascript String "" The unit to be displayed on top of the input field.
[clrUnitPosition]
Type: string
Default: "right"
 "left", "right" "right" Determines whether the unit is shown on the right or left side of the value.
[clrRoundDisplayValue]
Type: Boolean
Default: false
 true, false false Defines whether the value should be rounded or not.
