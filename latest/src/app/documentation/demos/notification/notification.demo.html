Notification is a component aimed to show user that particular action is done (for example, different success or error messages).

Usage

It’s aimed to notify user and show a message to him (error, warning, success, info). Notifications are important element for successful user workflow with application.

Notification Types

There are four different sub-types of notifications: error, warning, info, and success.

Error
Reserved for errors, malfunctions, as well as critical issues like license expiration.
Danger

Warning
Reserved for warnings: a message that needs the user attention and aknowledgment but might not cause errors.
Warning

Info
Provides info to users in context. Shouldn’t be overused to replace regular content.
Info

Success
Reserved to provide to a static persistent success message.
Success
