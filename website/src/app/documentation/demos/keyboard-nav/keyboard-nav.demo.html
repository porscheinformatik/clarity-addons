Two standalone directives — clrKeyboardNavCtrlArrow and clrKeyboardNavAltMnemonic — add keyboard-driven navigation to any Clarity navigation component (clr-tabs, clr-vertical-nav, or any custom host element) without changing the existing DOM structure.

Import

Import the directives individually or via ClrAddonsModule:

clrKeyboardNavCtrlArrow

Adds circular keyboard navigation using Ctrl+Arrow keys. Attach the directive to any Clarity navigation host and navigate without ever leaving the keyboard.

  • Ctrl+Left / Ctrl+Up — previous item (wraps to last).
  • Ctrl+Right / Ctrl+Down — next item (wraps to first).
  • Item order is captured once at page load and never changes.
  • Works globally — no need to focus the navigation host first.

API

Input Type Default Description
clrNavItemSelector string [clrVerticalNavLink], [clrTabLink] CSS selector for the navigable items inside the host element.
clrNavOrientation 'horizontal' | 'vertical' | 'both' 'both' Restricts which arrow directions are active. Use 'horizontal' for tabs, 'vertical' for sidebars.
clrNavDisabled boolean false Set to true to disable all navigation (e.g. while a modal is open).

clrKeyboardNavAltMnemonic

Assigns sequential numeric mnemonics (1, 2, 3, …) to navigation items and enables Alt-key-based navigation.

  • Mnemonics are assigned once at view initialisation and remain stable for the page lifetime.
  • While Alt is held the CSS class clr-kbd-mnemonics-visible is added to the host — styled badges appear on each item.
  • Digit keys pressed while Alt is held accumulate into a buffer.
  • Releasing Alt navigates to the matching item. Multi-digit mnemonics are supported: Alt + 1 + 0 → item 10.
  • When an item has a cds-icon, the icon is replaced in-place by a circle badge while Alt is held — no layout shift.
  • Alt+Arrow browser history navigation is never overridden.

API

Input Type Default Description
clrNavItemSelector string [clrVerticalNavLink], [clrTabLink] CSS selector for the navigable items inside the host element.
clrNavDisabled boolean false Set to true to disable all navigation (e.g. while a modal is open).

Both Directives Combined

Both directives can be applied to the same host element. Use either Ctrl+Arrow or Alt+digit to navigate — they operate independently and never interfere with each other.

The shared [clrNavDisabled] input deactivates both shortcuts at once when set to true, making it easy to guard against navigation while a modal or overlay is open.

Best Practices

  • Always set clrNavOrientation="horizontal" on clr-tabs to avoid conflicting with vertical shortcuts.
  • Bind [clrNavDisabled] to your overlay/modal open state to prevent accidental navigation while dialogs are open.
  • Both directives work with any focusable host element — not just Clarity components. Use clrNavItemSelector to target custom nav buttons.
  • The clrKeyboardNavAltMnemonic directive never overrides Alt+Arrow, preserving browser history navigation.
  • Mnemonics remain stable across the page lifetime. If items are added dynamically after initialisation, reinitialise the component to reassign mnemonics.
