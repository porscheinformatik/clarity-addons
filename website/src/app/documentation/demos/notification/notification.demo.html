Notifications are shown to inform the user of a particular event or completed action.

Angular Component

Notifications will be opened by a service, not by including the notification component in the markup. That way it's possible to use the same markup for multiple notification instances.

The service will be called with two parameters. The first one defines the content of the notification as a template or string. The second parameter defines the options for the notification, described below.

The service will return a ClrNotificationRef containing a promise result which will get resolved after the notification was closed.
