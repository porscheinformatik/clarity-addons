Keyboard operation directives extend the default Clarity form elements by providing additional functionality and behavior to enhance form interactions.

FocusFirstInvalidField Directive

The FocusFirstInvalidFieldDirective automatically focuses the first invalid form field when a form submit action is triggered. This improves user experience by immediately directing attention to fields that need correction, reducing the cognitive load on users.

When a form with this directive is submitted:

  • The directive listens to the form's submit event
  • It searches for the first element with the CSS class ng-invalid
  • If found, it focuses the first focusable child element (input, textarea, or select)
  • If the invalid element has no focusable children, it focuses the element itself
  • If all fields are valid, no focus change occurs

Independent Form Handling

Each form with the clrFocusFirstInvalidField directive operates independently. When multiple forms are present on a single page:

  • Each form manages its own validation state independently
  • Submitting one form only validates and focuses invalid fields in that form
  • Invalid fields in other forms are not affected
  • You can have as many forms as needed on a single page, each with independent directive handling

Supported Form Controls

The directive works with the following Clarity form elements:

  • Text inputs (clrInput)
  • Textareas (clrTextarea)
  • Select dropdowns (clrSelect)
  • Custom form controls with focusable child elements

Best Practices

  • Use this directive on forms that have validation requirements
  • Ensure form fields have proper ARIA labels for accessibility
  • Combine with Clarity's validation error messages for comprehensive feedback
  • Test with various form field types to ensure proper focus behavior
  • When using multiple forms on one page, be sure each form has unique field names to prevent conflicts

ControlEnterSubmit Directive

The clrControlEnterSubmit directive enables form submission using Ctrl+Enter (or Cmd+Enter on macOS). It provides a convenient keyboard shortcut for users to submit forms without leaving the input field.

Note: Normal Enter submission is prevented when this directive is applied. Only Ctrl+Enter (or Cmd+Enter) will trigger form submission.

Usage

Add the clrControlEnterSubmit directive to your form element. The directive listens for the key combination and triggers the form's submit event.

Optionally, you can pass a pre-translated tooltip string to the directive (e.g., [clrControlEnterSubmit]="'Ctrl+Enter to submit'"). If a string is provided, a tooltip with the shortcut hint will be displayed on the submit button. If no string is provided, no tooltip is shown.

Preventing Normal Enter Submission

When clrControlEnterSubmit is applied, pressing Enter alone will not submit the form. Only the Ctrl+Enter (or Cmd+Enter) combination will trigger submission. This prevents accidental form submissions and ensures keyboard shortcuts are used intentionally.
