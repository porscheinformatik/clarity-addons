Keyboard operation directives extend the default Clarity form elements by providing additional functionality and behavior to enhance form interactions.

FocusFirstInvalidField Directive

The FocusFirstInvalidFieldDirective automatically focuses the first invalid form field when a form submit action is triggered. This improves user experience by immediately directing attention to fields that need correction, reducing the cognitive load on users.

When a form with this directive is submitted:

  • The directive listens to the form's submit event
  • It searches for the first element with the CSS class ng-invalid
  • If found, it focuses the first focusable child element (input, textarea, or select)
  • If the invalid element has no focusable children, it focuses the element itself
  • If all fields are valid, no focus change occurs

Independent Form Handling

Each form with the clrFocusFirstInvalidField directive operates independently. When multiple forms are present on a single page:

  • Each form manages its own validation state independently
  • Submitting one form only validates and focuses invalid fields in that form
  • Invalid fields in other forms are not affected
  • You can have as many forms as needed on a single page, each with independent directive handling

Supported Form Controls

The directive works with the following Clarity form elements:

  • Text inputs (clrInput)
  • Textareas (clrTextarea)
  • Select dropdowns (clrSelect)
  • Custom form controls with focusable child elements

ControlEnterSubmit Directive

The clrControlEnterSubmit directive enables form submission via Ctrl+Enter globally. When Ctrl+Enter is pressed anywhere on the page, every form with this directive will be validated and submitted, regardless of focus. Native Enter submission for the focused form is preserved and unaffected. This allows both global keyboard shortcuts and standard submission behavior without interfering with validation logic.

Native Enter Submit: The directive does not override the default Enter submit for individual forms; users can submit a focused form as normal.

Global Ctrl+Enter Submit: When pressed, Ctrl+Enter submits every form with this directive on the page, regardless of focus.

Validation: Validation and error handling continue per normal Clarity usage.

Usage

Add the clrControlEnterSubmit directive to each form element you want to globally submit. The directive listens for Ctrl+Enter on the window and triggers submission for all such forms. Native Enter submit on an individual form remains supported and is not overridden. This allows a combination of both standard and enhanced global keyboard form handling.

Tooltip & Translation

Optionally, you can pass a pre-translated tooltip string to the directive (e.g., [clrControlEnterSubmit]="'Ctrl+Enter to submit'"). If a string is provided, a tooltip with the shortcut hint will be displayed on the submit button. If no string is provided, no tooltip is shown. Translation of the tooltip is not handled in within the directive.

Best Practices

  • Use clrControlEnterSubmit on any form requiring enhanced keyboard operation
  • Ensure forms are declared as clrForm
  • Ctrl+Enter triggers global multi-form submission, Enter triggers standard single form submit. Test combinations for undesirable side effects.
  • All validation logic is preserved and not altered by the directive.

Global Multi-Form Submission

When multiple forms with the clrControlEnterSubmit directive are present, pressing Ctrl+Enter submits every one of them at once, regardless of which form is currently focused. This provides a powerful global keyboard shortcut for rapid form submission across the entire page, enabling bulk actions or multi-form workflows.
