Usage

  • Don't mix core (webcomponents) and angular components inside a form, as they have different layout philosophies.
  • Always use forms. Don't display multiple labels and input fields outside of a form.
  • When adding subheaders inside a form, use h4 elements.
  • Control the width of labels and inputs by adding clr-col-* classes inside the clr-*-container.
  • Forms in a clr-view-edit-section should also be displayed using the below examples.
  • Make your forms responsive. Use the correct clr-col-* classes to make your forms fit every screen size.
  • Always use 1 column layouts on screen sizes below 768px. (md-breakpoint)

Submit form on enter

To enable form submit on enter, following prerequisites need to be done:

  • Add hidden submit button to your form to activate 'enter' key handling in browser (only needed if you not already have one)
  • React to submit of form
Submitted text (hit enter to submit): {{submittedText}}
Special case for updateOn: blur

Hitting the enter key doesn't trigger the blur event, so the form field is not up to date. To overcome this, blur the field manually.
