Clarity Addons provides six D3-powered chart components that are fully responsive, support CSS custom-property colours, legends, tooltips and SVG export. Import ClrChartsModule to use them all at once. Pay attention that the charts have their own entry point — @porscheinformatik/clr-addons/charts — that is separate from the main Addons module. This allows apps which don't use charts to avoid the extra bundle weight of D3 and its dependencies.

Module Setup

Add ClrChartsModule to your NgModule imports (or import the six standalone-wrapped components individually):

Shared Inputs (all charts)

Input Type Default Description
loading boolean false Shows a skeleton placeholder instead of the chart.
showLegend boolean true Toggles the colour-keyed legend below the chart.
showExportButton boolean false Displays a download button that exports the chart as SVG.
exportButtonTitle string Export Title of the export button.
exportFilename string "<chart-type>" File name (without extension) used for the SVG export.
tooltipOrientation 'top' | 'bottom' 'top' Where the tooltip anchor appears relative to the hovered element.
noItemsMessage string built-in Message shown in the info-alert when the data array is empty.

Color Handling

Every chart component accepts colors on individual data items (or series). The color property accepts two formats — the internal toChartColor() utility resolves both automatically. You can use all colors defined in Clarity or Clarity Addons.

Format Example value Resolved CSS value When to use
Hex / RGB / HSL '#e57200'  ·  'rgb(229,114,0)' passed through as-is Fixed brand colors or one-off overrides
CSS Custom Property '--clr-color-action-600' var(--clr-color-action-600) Design-token colors that automatically adapt to theme changes (light / dark mode)
CSS custom properties are detected by a leading --. You pass the name without the var(…) wrapper — the chart resolves it internally.

Bar Chart <clr-bar-chart>

Renders vertical or horizontal bars. Each item requires a unique key, a display label, a numeric value, and an optional color (hex or --css-custom-property).

Area Chart <clr-area-chart>

An area chart is a line chart with the region below each line filled. Use [areaOpacity] to keep overlapping areas readable.

Combo Chart <clr-combo-chart>

Overlays one or more line series on top of grouped/stacked bar series sharing the same X axis and Y scale. Ideal for comparing actuals vs. targets.

Inputs

Input Type Default Description
barSeries ComboBarSeries[] [] Bar series. Each item's data requires x, xLabel and value.
lineSeries ComboLineSeries[] [] Line series overlaid on the bars.
xAxisLabel / yAxisLabel string '' Optional axis description labels.
yLineAxisLabel string '' Optional label for the right Y axis (line scale).
yBarMax number | undefined undefined Fixes the maximum value of the left (bar) Y axis. When omitted the scale is calculated automatically from the data.
yLineMax number | undefined undefined Fixes the maximum value of the right (line) Y axis. When omitted the scale is calculated automatically from the data.

Example

Fixed Y Axis Max Values (yBarMax / yLineMax)

Use [yBarMax] and [yLineMax] to pin the upper bound of either Y axis to a fixed value. This is especially useful when comparing multiple charts side-by-side on a consistent scale.

Clipping – Values Exceed the Maximum

In the following example yBarMax="200" and yLineMax="100" are set, but some data points exceed those limits. Bars and lines are clipped at the maximum boundary; the tooltip still shows the real value.

Funnel Chart <clr-funnel-chart>

Visualises a multi-stage pipeline. Each stage can optionally be broken into coloured sections. Two orientations are available: 'default' (horizontal bars) and 'centered' (trapezoid funnel).

Inputs

Input Type Default Description

barGap number 4 Vertical gap between bars in pixels.
chartLabels Partial<FunnelChartLabels> built-in Override the "Total" and "All" label strings (e.g. for i18n).

Adding Additional Charts

All chart components are built on a shared ChartBase abstract class and registered through ClrChartsModule. Follow the steps below to add a new chart type that integrates seamlessly with the existing skeleton, tooltip, legend, and export infrastructure.

1. Create the component files

Create a new folder under src/clr-addons/charts/<my-chart>/ containing:

  • my-chart.component.ts — the component class
  • my-chart.component.html — the template
  • my-chart.component.scss — optional styles

2. Extend ChartBase

ChartBase<TSelected> wires up the container resize listener, skeleton state, tooltip position signal, and SVG export reference. Your component only needs to implement updateChart(), which is called automatically after the view is ready and on every subsequent resize.
