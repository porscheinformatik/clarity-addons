All pages in an application should follow a common layout structure to ensure a consistent user experience across applications.

Base Pagelayout

The Base Pagelayout provides the most fundamental navigation and content elements.

The Base Pagelayout includes the following components:

  • Main Navigation
  • Pagetitle
  • Content Area

ATTENTION! Following DOM structure is vital for a correctly working page layout. They MUST be DIRECT children of each other!

  • .main-container or .u-main-container
    • .content-header
    • flowbar (optional)
    • .content-container
      • .content-area
      • vertical nav (optional)
      • .content-area-footer (optional)

Further components may complete the pagelayout as needed:

  • Back Button
  • Command bar
  • Sidebar
  • Content-Panel
  • Action-Panel

Command Bar

  • The Command Bar comprises the most common users tasks.
  • The most important ones are shown, additional ones are accessible in an overflow-menu.
  • Other commands, either less common ones or where the proximity to an ui element is crucial, are provided within the content area.
  • Consistent button order in command-bar:
    • The history-button is the left-most button, followed by
    • Action Buttons, and
    • Optional: the action-panel-button is the right-most button in the command-bar

Action Panel

  • Use when lot of actions are required
  • Group similar actions using Captions and Spacer.
  • Don't use when only a few actions are required, consider using Command bar then.

Following example shows how to use the action panel.

Add a button to show/hide the action panel. This can be done by calling the toggle method of the action panel component.

Flow Bar Layout

The Flow Bar Layout extends the Base Page Layout by adding a wizard-like navigation control.

This layout that can be used to highlight the current progress a user is making throughout multiple steps. The Flow Bar allows the user to navigate between steps, by either clicking on a specific step or using the previous or next button.

The Flow Bar Layout requires the ClrFlowBar and can optionally also contain the Sticky Footer Layout.

Usage

Use this layout if you want to guide the user through a sequence of multiple steps. Each step requires the user to perform a complex task like filling out a form. Don't use this layout if the user's steps are mutually exclusive; in this case use Tabs instead.

When implementing a flow bar on your page, you will need to insert a clr-flow-bar right below your content-header but above your content-container.

Following inputs & outputs are available:

  • [clrSteps] - an array of FlowBarStep. You can extend/implement this interface to save even more information in your steps.
  • [(clrActiveStep)] two way binding of the active step.

You can use the @ViewChild() annotation to get a reference of your ClrFlowBar, which will allow you to call its methods:

  • previous() - open the previous step.
  • next() - opens the next step.
  • isPreviousAvailable() - returns true if the previous step can be opened.
  • isNextAvailable() - returns true if the next step can be opened.
  • isLastStep() - returns true if the current step is the last.

Sidebar Pagelayout

The Sidebar Pagelayout enhances the Base Pagelayout with a vertical navigation sidebar.

Design Guidelines

The Sidebar Pagelayout includes the following components:

  • Main Navigation
  • Pagetitle
  • Content Area
  • Vertical Nav (Sidebar)

Futher components may complete the pagelayout as needed:

  • Command Bar

Use when

Use the Sidebar Pagelayout when you have:

  • Lots of content on one page, that can be junked in logical separated pages
  • If you need a summary or overview-page, this should be the first element in the sidebar

Sticky Footer Layout

The Sticky footer represents a bar at the bottom of your page which is always visible - scrollable content is hidden behind the sticky footer.

The Sticky footer can be used for e.g. providing navigational buttons for a wizard (See also Flow Bar Layout).

Usage

Use this component when you want to display permanently visible elements on the bottom of a page. It works particularly well in combination with the Flow Bar Layout to provide navigational controls. When the Sticky Footer contains buttons for navigation, keep in mind to place them on the right hand side to follow a logical Z Pattern.

Do

Place buttons leading to next step on the right. Optional information about the user's actions are placed on the left side.

Don't

Don't put navigational buttons on the left side.

When implementing a sticky footer on your page, you will want to insert a div with class content-area-footer right below your content-area div. This can be combined with a content panel layout, see the code example of the flow bar layout for more information.

Full Screen Dialog

The Full Screen Dialog is an alternative to the default modal dialog.

The Full Screen Dialog looks and feels like any other page though it is conceptually a modal dialog. This means that a full screen dialog can have any page layout. From a technical point of view, full screen dialogs are shown in a router outlet above the current page. This eliminates the loading time which usually occurs while navigating from page to page.

Design Guidelines

A Full Screen Dialog usually contains mutliple input elements and Save and Cancel options. Place these call to action buttons on the left side of the bottom of the page. Just like you would in a regular form. Optionally the dialog can also have a Back-button. Provide a confirmation prompt if the user has already entered data and Back was selected. Note that this prompt is not required for the Cancel-button, since the user deliberately choose Cancel to discard the input.

Use when

Use the Full Screen Dialog if:

  • You want to display a complex input form that requires Save and Cancel options
  • You need to show additional modal dialogs. In general you should not show another modal over a modal, but you can open a modal over a Full Screen Dialog.
  • The dialog's content would require too much scrolling in a modal dialog. Try to avoid scrolling in modal dialogs.
