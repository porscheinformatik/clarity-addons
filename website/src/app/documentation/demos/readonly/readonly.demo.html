Usage

  • Use Readonly when the information is valid but should not be editable.
Difference between Enabled, Disabled and Readonly
  • enabled: The value of the input can be edited
  • disabled: The value of the input is invalid in the current context, not applicable and can not be edited.
  • readonly: The value of the input is valid in the context, but cannot be edited.
Example
 The store location of an item
 It is enabled when the user has the right to edit the store location.
 It is disabled when the item has been moved to an external storage-space. The previous entered store location is not valid any more. It is readonly when the current user should only be able to find the item, but not after its store location.
Do
Do display valid information that is not editable in readonly state. For example due to restricted rights.
