The Summary Area is a grid-based component used to display a compact overview of key information at the top of a page. It presents label-value pairs in a structured, scannable layout ideal for showing entity details, status information, or metadata at a glance.

Use When

  • You need to provide a quick overview of essential information about an entity (e.g., customer, order, vehicle).
  • Displaying read-only metadata or status information at the top of a detail page.
  • Users need to scan multiple pieces of related information quickly.
  • The information is static or updates infrequently during a session.
  • You want to show up to 15 items in a structured 5-column × 3-row grid layout.

Do not use

Avoid using this component in any of the following cases:

  • For data that requires frequent editing – use the View-Edit Section instead.
  • For displaying large amounts of data – use a Data Grid instead.
  • For form inputs or editable fields – use standard Clarity form components.
  • When you have more than 15 items – consider restructuring the information or using tabs/sections.
  • For navigation purposes – use the appropriate navigation components.

Behavior

The Summary Area displays information as label-value pairs in a responsive grid layout with up to 5 columns and up to 3 rows. Summary Items are arranged column-first, filling columns from top to bottom before moving to the next column. On smaller screens, the number of columns automatically reduces to fit the available space.

Collapsible State

The Summary Area can be collapsed and expanded using the clr-summary-area-toggle component. When collapsed, only the toggle button is visible, allowing users to maximize content space while keeping summary information accessible.

Loading, Error, and Warning States

Both the Summary Area and individual Summary Items support loading, error, and warning states. These states provide visual feedback to users about data availability and potential issues. All of those states have to be provided explicitly to the Summary Area or a Summary Item via their respective inputs.

  • Loading: Shows a spinner with optional customized text while data is being fetched.
  • Error: Displays an error indicator with optional customized text and optional clickable action link .
  • Warning: Shows a warning indicator with optional customized text and optional clickable action link .

Empty Values

Items without values can either show a placeholder dash (—) or be hidden entirely, controlled via the [showOnEmptyValue] input. Alternatively, an item can also define an edit configuration via the [editConfig] input to show a custom action link when no value is present.

Clickable Values

Summary item values can be made clickable with the [clickable] input for navigation or triggering actions. When a value is clickable, it appears as a link and emits a (clicked) event.

Copy to Clipboard

Items can be configured with the input [valueCopyable] to show a copy button that copies the value text to the clipboard. This is useful for IDs, codes, or other values users might need to copy.

Technical Limitations

  • Maximum 5 columns supported.
  • Maximum 3 rows supported (items beyond 15 will not be visible).
  • Icons are only allowed as the first value component in a Summary Item.
  • A clr-summary-item-value can have either an icon OR a text value, not both.
  • Copy functionality only works with clr-summary-item-value components, not projected content.

Model Interfaces

The Summary Area and Summary Item components use specific model interfaces to configure loading, error, warning, and edit states. Below is a detailed description of each interface and how their properties affect the component behavior.

ClrSummaryAreaLoading

Used to display a loading indicator with an optional custom message on the Summary Area.

Property Type Description
active boolean Required. When true, the Summary Area displays a loading spinner and hides all summary items. Loading state takes highest priority over error and warning states.
text string Optional custom text displayed next to the loading spinner. Defaults to "Loading..." if not provided.

ClrSummaryAreaError

Used to display an error state with an optional clickable action link on the Summary Area. The error state is only shown when the loading state is inactive.

Property Type Description
active boolean Required. When true and loading is inactive, the Summary Area displays an error indicator and hides all summary items. Error state takes priority over warning state.
text string Optional custom error message displayed next to the error icon. Defaults to "Error" if not provided.
linkText string Optional text for the action link displayed after the error message. Only shown when click callback is also provided.
click () => void Optional callback function invoked when the action link is clicked. Typically used to retry loading or navigate to an error details page.

ClrSummaryAreaWarning

Used to display a warning state with an optional clickable action link on the Summary Area. The warning state is only shown when both loading and error states are inactive.

Property Type Description
active boolean Required. When true and both loading and error states are inactive, the Summary Area displays a warning indicator and hides all summary items.
text string Optional custom warning message displayed next to the warning icon. Defaults to "Warning" if not provided.
linkText string Optional text for the action link displayed after the warning message. Only shown when click callback is also provided.
click () => void Optional callback function invoked when the action link is clicked.

ClrSummaryAreaColumns

A type alias for the allowed number of columns in the Summary Area grid.

Type Description
1 | 2 | 3 | 4 | 5 The Summary Area supports a maximum of 5 columns. The actual number of columns displayed is calculated dynamically based on the number of items, the configured rows, and the available screen width.

ClrSummaryAreaRows

A type alias for the allowed number of rows in the Summary Area grid.

Type Description
1 | 2 | 3 The Summary Area supports a maximum of 3 rows. Combined with the 5 column limit, this means a maximum of 15 summary items can be displayed. Items beyond this limit will not be visible.

ClrSummaryItemLoading

Used to display a loading indicator with an optional custom message on an individual Summary Item.

Property Type Description
active boolean Required. When true, the Summary Item displays a loading spinner instead of its value. Loading state takes highest priority over error and warning states at the item level.
text string Optional custom text displayed next to the loading spinner. Defaults to "Loading..." if not provided.

ClrSummaryItemError

Used to display an error state with an optional clickable action on an individual Summary Item. The error state is only shown when the loading state is inactive.

Property Type Description
active boolean Required. When true and loading is inactive, the Summary Item displays an error indicator instead of its value. Error state takes priority over warning state.
text string Optional custom error message. Defaults to "Error" if not provided.
click () => void Optional callback function invoked when the error text is clicked. When provided, the error text becomes a clickable link.

ClrSummaryItemWarning

Used to display a warning state with an optional clickable action on an individual Summary Item. The warning state is only shown when both loading and error states are inactive.

Property Type Description
active boolean Required. When true and both loading and error states are inactive, the Summary Item displays a warning indicator instead of its value.
text string Optional custom warning message. Defaults to "Warning" if not provided.
click () => void Optional callback function invoked when the warning text is clicked. When provided, the warning text becomes a clickable link.

ClrSummaryItemEditConfig

Used to display an edit link when the Summary Item has no content. This is useful for providing an action to add or edit data when the value is empty.

Property Type Description
enabled boolean Required. When true, an edit link is shown if no content is projected into the Summary Item and no loading, error, or warning state is active. When set, this overrides the [showOnEmptyValue] input behavior.
text string Optional custom text for the edit link. Defaults to "Edit" if not provided.
click () => void Required. Callback function invoked when the edit link is clicked. Typically used to open an edit form or dialog.

State Priority

Both the Summary Area and Summary Item components follow a priority hierarchy when determining which state to display:

  1. Loading — Always takes highest priority when active
  2. Error — Shown when loading is inactive
  3. Warning — Shown when both loading and error are inactive
  4. Edit (Summary Item only) — Shown when no state is active and no content is present
  5. Content / Empty Value — Normal display when no state is active

Basic Summary Area

Summary Item with Icon

Icons can be placed as the first value element to provide visual context. Note that icons can only be used as the first clr-summary-item-value child element in an clr-summary-item element.


Clickable Values

Values can be made clickable to navigate or trigger actions. Both icons and text values can be independently clickable.


Multiple Values

A Summary Item can contain multiple text values, displayed in sequence.


Tooltip and Long Text

Long text values are automatically truncated with ellipsis and display a tooltip with the full value. Custom tooltips can be set for additional context. When a value overflows, its text takes precedence for the tooltip content over any custom tooltip.


Empty Value Handling

Items without values can show a placeholder or be hidden entirely. Use [showOnEmptyValue]="true" to show a long dash placeholder or [showOnEmptyValue]="false" to hide the item.


Edit Configuration

Empty items can display an edit link when the edit configuration is enabled. This is useful for prompting users to add missing data.


Copy to Clipboard

Enable the copy button to allow users to easily copy values to the clipboard. The copy button shows a success indicator when clicked. When copying the value of a summary item with multiple values, the values are concatenated with spaces in between (same as they are displayed). Only values defined with clr-summary-item-value components can be copied. Projected content cannot be copyied via the copy button.

Collapsible Summary Area

Use the clr-summary-area-toggle component to let users collapse and expand the Summary Area. Ensure that ClrSummaryAreaStateService is provided for the summary area component. For independent collapse and expand behavior across multiple summary areas in your application, assign a unique value to the [localStorageKey] input for each pair of clr-summary-area-toggle and clr-summary-area. This prevents the collapse state of one summary area from affecting another.


Custom Projected Content

You can project custom content into a Summary Item for more complex displays. Note that copy functionality is not available for projected content.
