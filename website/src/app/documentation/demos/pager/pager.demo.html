Dot Pager

The Dot-Pager is a basic component that can be used to show a simple pagination bar.

Usage

  • Use the Dot-Pager component whenever you have to show data over multiple pages, e.g. a list of Cards.
  • Do not use the Dot-Pager component when you are showing your data in a clr-datagrid, instead use the existing clr-dg-pagination component.

Code & Examples

Following interactions are available:

  • To define the pages add a [clrPages] input to the component.
  • To define the current page add a [clrCurrentPage] input to the component.
  • To react to page changes, bind to the event (clrCurrentPageChange).

Generic Pager

The Generic Pager is a basic component that can be used to show a simple pagination bar.

Usage

  • Use the Generic Pager component whenever you have to show data over multiple pages, e.g. a list of Cards.
  • Do not use the Generic Pager component when you are showing your data in a clr-datagrid, instead use the existing clr-dg-pagination component.

Code & Examples

Following interactions are available:

  • To define the page size add a [clrPageSize] input to the component.
  • To define the number of total items add a [clrTotalItems] input to the component.
  • To react to page changes, bind to the event (clrPageChange).
  • Optionally you can set the current page by adding a [clrPage] input to the component.

Paged Search Result List

The Paged Search Result List component is a basic component used to display a list of generic data including a pager above and below the list.

Style

You can style the list any way you want - this component allows you to use a custom template to show all items of your list. The custom template can range from a single line representing text to one or more completely new angular components.

Usage

To set a template for each item of your list simply add a ng-template inside the clr-paged-search-result-list. By defining let-item="item" on your ng-template you can get the context of each individual item.
