Clarity Addons extensions for the native Clarity clr-tooltip component.

The native Clarity tooltip content renders at a fixed width (determined by [clrSize]). The clrTooltipFitContent attribute directive overrides this by applying width: fit-content !important directly on the tooltip content element, so the bubble only takes as much space as its text needs.

Summary of Options

Directive / Input Type Default Description
clrTooltipFitContent
Type: boolean
Default: true
 boolean true Attribute directive applied to <clr-tooltip-content>. When true (the default when the attribute is present without a binding), sets width: fit-content !important on the tooltip bubble. Pass [clrTooltipFitContent]="false" to disable it conditionally.

Real-world usage in clr-copy-to-clipboard

The clr-copy-to-clipboard component applies clrTooltipFitContent internally on its own clr-tooltip-content, so the "Copy to clipboard" bubble is always sized to its text rather than stretching to a fixed Clarity size. No extra input is needed on the component itself.
